# 性能优化建议

1. 图片使用 `lazy-load`，并在正式环境改用云存储压缩图或 WebP。
2. 首页菜单按分类分页加载，当前脚手架默认 `pageSize = 8`。
3. 推荐、订单统计、经营面板优先走云函数聚合，减少小程序端多次往返。
4. 购物车与常用用户数据做本地缓存，减少重复请求与页面切换闪烁。
5. 管理后台列表在真实业务中建议按状态分页并增加搜索条件，避免全集合扫描。
6. 对按钮点击、提交下单增加客户端防重与服务端幂等，降低重复请求压力。
7. 建议为首页增加骨架屏，为订单页增加增量刷新策略，优化弱网体验。
8. 如菜品图片较多，可结合云存储 CDN、自定义尺寸裁剪和分级加载策略。
