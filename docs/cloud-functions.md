# 云函数说明

## 1. `loginUser`

### 职责

- 通过 `OPENID` 查找用户
- 已存在用户则更新昵称、头像和登录时间
- 不存在则创建新用户

### 输入

```json
{
  "loginCode": "微信登录 code",
  "profile": {
    "nickname": "昵称",
    "avatarUrl": "头像地址"
  }
}
```

### 输出

```json
{
  "code": 0,
  "data": {
    "_id": "user-id",
    "role": "customer"
  }
}
```

## 2. `placeOrder`

### 职责

- 校验幂等 token
- 校验菜品是否上架、库存是否足够
- 事务创建订单
- 扣减库存与增加销量
- 更新用户订单统计

### 特点

- 使用事务保证订单与库存一致
- 服务端重新计算金额，不信任前端总价

## 3. `updateOrderStatus`

### 职责

- 校验当前用户是否有权限操作订单
- 校验状态流转是否合法
- 取消订单时回滚库存与销量
- 写入状态时间线

## 4. `manageDish`

### 职责

- 校验管理员权限
- 新增菜品
- 编辑菜品
- 上下架菜品
- 设置首页推荐

## 5. `adminDashboard`

### 职责

- 统计订单总量
- 统计总营收
- 统计待处理订单数
- 统计上架菜品数
- 输出热销 TOP 列表

## 6. `recommendDishes`

### 职责

- 基于已完成订单构建用户偏好向量
- 使用协同过滤计算相似用户
- 输出推荐菜品
- 若推荐不足则降级为热销补齐

## 7. 设计原则

所有云函数都遵循：

- 单一职责
- 清晰输入输出
- 权限校验明确
- 不把多个大业务堆在一个超大函数中
